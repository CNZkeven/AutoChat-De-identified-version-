# 评价型智能体测试用例

## 用例 1：作业重复错误（严厉提醒）
- 目的：验证错题本调用与严厉提醒
- 输入：
  - 角色：高校学生
  - 提问："老师，这是我这次的 PID 控制实验报告，请帮我批改。"
- 侧重点：
  - 是否调用 check_error_book + profile basic
  - 是否指出“重复错误”并要求修改
  - 是否给出暂定分数与检查清单

## 用例 2：作业改正旧错（鼓励）
- 目的：验证对“改正旧错”的积极反馈
- 输入：
  - 角色：高校学生
  - 提问："我这次把单位都补齐了，能帮我看看实验报告吗？"
- 侧重点：
  - 是否肯定改进
  - 是否给出可进一步提升的建议

## 用例 3：基础问题 + 年级差异
- 目的：验证根据年级调整语气
- 输入：
  - 角色：高校学生（大三）
  - 提问："老师，什么是 PID 控制？"
- 侧重点：
  - 是否调用 profile basic
  - 是否强调高年级需复习基础

## 用例 4：能力画像与成长路径
- 目的：验证能力评估与引导提问
- 输入：
  - 角色：高校学生
  - 提问："我最近控制理论总是做不好，问题在哪？"
- 侧重点：
  - 是否调用 profile academic
  - 是否给出能力评估（如理论 C/实践 B）
  - 是否提出启发式问题

## 用例 5：信息不足（追问）
- 目的：验证缺少关键信息时的追问
- 输入：
  - 角色：高校学生
  - 提问："老师帮我看看作业哪里错了。"
- 侧重点：
  - 是否追问课程/作业类型/年级等
  - 是否避免直接给结论

## 用例 6：按学期成绩查询
- 目的：验证 list_scores 调用与学期筛选
- 输入：
  - 角色：高校学生
  - 提问："帮我看一下 2024-2025-1 学期的成绩概览。"
- 侧重点：
  - 是否调用 fetch_dm_student_academic_data(action="list_scores", term=...)
  - 是否基于工具结果给出评价

## 用例 7：单门课程成绩
- 目的：验证课程筛选与成绩获取流程
- 输入：
  - 角色：高校学生
  - 提问："自动控制原理这门课我的成绩怎么样？"
- 侧重点：
  - 是否先 list_course_offerings 获取 offering_id
  - 是否调用 course_grades

## 用例 8：课程目标达成
- 目的：验证课程目标与达成度查询
- 输入：
  - 角色：高校学生
  - 提问："自动控制原理的课程目标达成情况如何？"
- 侧重点：
  - 是否先 list_course_offerings 获取 offering_id
  - 是否调用 course_objectives + course_achievements
