# 任务型智能体测试用例

## 用例 1：新任务 + 截止期限（课程设计）
- 目的：验证计划生成 + 课程信息查询
- 输入：
  - 角色：高校学生
  - 提问："我需要在两周内完成控制系统课程设计，帮我拆解任务并安排进度。"
- 侧重点：
  - 是否调用 generate_plan + profile basic
  - 是否调用 curriculum 查询
  - 是否输出清单/时间规划/风险提醒

## 用例 2：多任务合并与遗留提醒
- 目的：验证任务合并与优先级排序
- 输入：
  - 角色：高校学生
  - 提问："上周文献综述没写完，本周还要准备答辩PPT和实验数据分析，帮我排一下。"
- 侧重点：
  - 是否生成可执行清单
  - 是否提示优先级与时间节奏

## 用例 3：进度打卡
- 目的：验证里程碑记录与下一步提示
- 输入：
  - 角色：高校学生
  - 提问："我已经完成第一章背景与意义，下一步怎么做？"
- 侧重点：
  - 是否调用 log_milestone
  - 是否给出下一节点与今日行动

## 用例 4：执行受阻（风险与缓冲）
- 目的：验证风险识别与缓冲方案
- 输入：
  - 角色：高校学生
  - 提问："实验平台坏了，实验进度可能延误，怎么办？"
- 侧重点：
  - 是否给出风险与备选方案
  - 是否调整节奏

## 用例 5：提醒与周计划
- 目的：验证提醒类请求与周计划拆解
- 输入：
  - 角色：高校学生
  - 提问："这周任务很多，帮我安排一个每天的待办清单。"
- 侧重点：
  - 是否给出可执行的日程清单
  - 是否明确当前节点
