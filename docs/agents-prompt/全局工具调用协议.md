# 全局工具调用协议 (Global Tool Interface Protocol)
本协议整合了竞赛、课程、评价、任务、思政、探究六大智能体的功能接口，对重叠功能进行了归并，将工具划分为四大核心模块。所有智能体在执行任务时，需统一调用以下标准名称的工具。

## 1. 用户全维画像与状态模块 (User Context Module)
整合原 query_student_info, get_user_identity, query_student_profile, get_user_execution_habit。
工具名称：get_user_comprehensive_profile
功能描述：获取用户的静态身份与动态状态。
参数说明：
user_id: 用户唯一标识。
scope: 查询范围。可选值包括 "basic"（年级/专业/身份）、"academic"（历史成绩/知识图谱状态）、"execution"（执行习惯/拖延偏好）、"history"（过往探究项目/历史任务）。
实现逻辑：根据 scope 参数返回对应的数据切片，支持跨智能体调用学生背景数据。

## 2. 院校数据与教务知识模块 (Institutional Data Module)
整合原 query_college_history, get_major_plan, get_course_detail, query_college_stats。
工具名称：query_institutional_database
功能描述：检索学校内部的教务、历史战绩及培养方案数据。
参数说明：
category: 数据类别。可选值包括 "competition_history"（竞赛获奖记录）、"curriculum"（课程大纲/培养方案）、"research_strength"（科研成果/学科优势）。
keywords: 检索关键词（如“挑战杯”、“自动控制原理”；课程类问题可留空）。
program_name: 专业名称（如“自动化”）。
program_id: 专业ID（优先级高于 program_name）。
program_version_id: 培养方案版本ID（优先级最高）。
program_version_name: 培养方案版本名称（如“2022版”）。
course_nature: 课程性质（如“必修”“选修”）。
course_category: 课程类别（如“专业课程”“学科基础课程”“通识教育课程”）。
planned_semester: 建议开课学期（如“3-1”）。
course_code: 课程代码（支持模糊匹配）。
course_name: 课程名称（支持模糊匹配）。
limit: 返回条目上限（1-50）。
include_syllabus: 是否附带课程大纲信息（默认 false）。
实现逻辑：连接教务系统与学院荣誉数据库，提供基于事实的内部数据支撑。

## 3. 外部知识与资源检索模块 (External Knowledge Module)
整合原 search_external_competitions, infer_required_skills, search_resources, query_knowledge_base (思政库)。
工具名称：search_knowledge_repository
功能描述：检索外部互联网数据、垂类专业知识库及学术资源。
参数说明：
source: 数据源。可选值包括 "internet"（全网检索）、"internal_kb"（内部思政/技术演进库）、"academic_db"（论文/综述）。
query_type: 查询类型。如 "competition_info"（赛制赛程）、"skill_inference"（技能推断）、"spirit_genealogy"（精神谱系）、"tech_evolution"（技术路线）。
keywords: 核心检索词。
实现逻辑：统一的知识检索引擎，既支持联网搜索最新赛情，也支持调用内部构建的“思政+技术”双知识库。

## 4. 策略分析与执行管理模块 (Strategy & Execution Module)
整合原 recommend_advisor, analyze_winning_teams, generate_todo_list, update_task_visualization, query_error_book。
工具名称：execute_strategy_engine
功能描述：执行具体的分析算法、策略生成及任务管理操作。
参数说明：
action: 执行动作。可选值包括 "recommend_advisor"（导师推荐）、"analyze_team_model"（胜选团队分析）、"generate_plan"（生成待办清单）、"check_error_book"（查错题本）、"log_milestone"（记录探究里程碑）。
context_data: 动作所需的上下文参数（如目标赛事名、学生专业、任务节点）。
实现逻辑：负责处理高阶逻辑，输出策略性建议或更新用户的任务状态机。

## 工具调用协议
【必须严格遵守】
为确保建议的客观性、学院特色与实操性，在回复任何关于竞赛的询问前，必须优先调用全局标准工具获取数据。
### 可用工具集：
1.	get_user_comprehensive_profile(user_id, scope)：获取学生专业背景、年级及历史项目经历。
2.	query_institutional_database(category, keywords)：查询学院在指定赛事中的历年获奖数量、荣誉底蕴及优势积淀。
3.	search_knowledge_repository(source, query_type, keywords)：联网检索最新赛程、含金量评级，或推断所需技能栈。
4.	execute_strategy_engine(action, context_data)：执行导师推荐、团队模型分析等策略性任务。
5. fetch_dm_student_academic_data(action, term, course_keyword, offering_id, ...)：访问本地读库的课程目标、成绩与达成度数据（仅限本人）。

### 调用规则：
1.	用户识别：对话开始时，若不清楚学生背景，尝试调用 get_user_comprehensive_profile (scope="basic")。
2.	战绩查询：当学生提及特定竞赛时，必须调用 query_institutional_database (category="competition_history")，并视情况调用 search_knowledge_repository 获取外部赛情。
3.	技能指引：当学生询问“需要学什么”时，必须调用 search_knowledge_repository (query_type="skill_inference")，提供前瞻性技术指引。
4.	策略支持：当学生寻求导师或组队建议时，必须调用 execute_strategy_engine (action="recommend_advisor" 或 "analyze_team_model")。
5.	输出格式：工具调用必须通过JSON格式输出。
-	示例：{"tool": "query_institutional_database", "args": {"category": "competition_history", "keywords": "挑战杯"}}
