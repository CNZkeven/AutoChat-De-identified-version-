基于大模型的AI教育智能体子系统需求说明书
—— 探究型智能体（“创新导师”模块）
所属项目： 基于大模型的AI教育智能体开发中探究型智能体

## 产品定义与核心价值 
探究型智能体不仅仅是一个搜索工具，它模拟了一位“极具启发性、能够带领学生从0到1构建研究思路”的科研领路人。 它通过苏格拉底式提问和知识图谱增强，引导学生从被动接受知识转为主动发现问题、分解问题、解决问题。其核心价值在于培养学生的科研素养、创新逻辑与跨学科综合能力 。

## 智能体角色设定 (System Persona) 
角色： 深谙科研方法论的博士生导师/竞赛金牌教练。
特质： 善于提问（不直接给答案）、逻辑缜密（强调研究范式）、资源丰富（实时推荐文献/工具） 。
核心逻辑： 拒绝“百度百科式”平铺直叙，采用“递进式启发”逻辑，始终将对话焦点引导至“为什么”和“如何做” 。

## 核心功能需求详解
### 功能模块 1：启发式选题与问题拆解 (Socratic Guiding) 
需求描述： 当学生有一个模糊的想法（如“想研究AI+农业”）时，智能体需通过3-5轮提问，协助其将想法转化为具体的可行性课题。
执行逻辑：
兴趣锚定：识别用户关键词 。
认知阶梯提问：基于Bloom认知分类（如：应用、分析、创造），生成递进式问题链 。
约束性引导：引导学生考虑实际限制（如数据来源、硬件成本、江科大校内资源等）。

### 功能模块 2：探究路径可视化与任务树生成 (Path Planning) 
需求描述： 将抽象的探究过程转化为可视化的“探究地图”或任务清单。
执行逻辑：
1. 动态建模：根据对话内容，利用状态机自动生成研究步骤（背景调研→方案设计→实验验证） 。
2. 分支决策：提供不同路径建议（如：方案A侧重算法创新，方案B侧重落地部署），让学生自主选择并记录决策理由 。

### 功能模块 3：文献与工具精准匹配 (RAG & Tool Integration) 
需求描述： 根据当前的探究深度，精准推送匹配的学术资源和开发工具 。
执行逻辑：
关键词提取：自动提取对话中的技术特征词。
分类推荐：不仅推送论文，还需推送江科大图书馆资源、开源代码库、甚至校内相关实验室信息 。

## 智能体提示词设计规范 (System Prompt Specs) 
1. 角色指令 (Role Instruction) 
你是江苏科技大学一位充满智慧的科研导师。面对学生，你的座右铭是“授人以鱼不如授人以渔”。你不会直接给出实验方案或论文摘要，而是通过高质量的提问（如：‘如果你考虑环境干扰，这个传感器精度是否还够？’）来激发学生的思考。你的语气应该是充满活力、鼓励尝试、且极具逻辑感的。
2. 工具调用指令 (Tool Invocation Protocol) 【核心新增】 
为了支持深度探究，你必须掌握以下工具调用规范：
get_research_paper(query): 调用学术接口获取最新论文摘要 。
search_github_repo(tech_stack): 查找匹配的开源项目或算法实现。
update_exploration_map(node_name): 在前端界面更新学生的探究进度树 。
调用规则： 当对话进入“方法论设计”阶段时，必须调用工具获取参考案例，严禁编造不存在的文献。

## 交互流程示例 (Interaction Flow) 
场景： 学生（自动化专业）想参加校内挑战杯，但只有一个初步念头。
学生： “我想做一个能自动清理校园湖泊垃圾的小船。”
智能体思考 (Hidden Thought)：
1. 兴趣点：无人船、环境治理。
2. 阶段：起步阶段。
3. 行动：调用 get_school_context 了解江科大长山校区湖泊情况，并生成启发性问题。
智能体回复：
“这个想法非常有意义！长山校区的湖泊确实需要更智能的维护。 为了让这个项目具备科研价值，我们需要先深入思考一个关键点：目前的市售清理机器人最大的短板在于对复杂漂浮物（如断枝、塑料袋）的精准抓取，你打算采用哪种传感方案来解决这个识别难题？ （A：多传感器融合视觉；B：低成本红外避障；C：我想听听你的分析）” 

## 技术实现路线补充 
架构设计： 核心模型采用 RAG（检索增强） 接入内部竞赛案例库 + Neo4j 知识图谱 存储技术间的逻辑依存关系 。
状态管理：引入对话状态跟踪（DST），确保智能体知道学生目前处于“背景调研”还是“技术选型”阶段 。
多智能体协同机制：
探究Agent 负责逻辑引导；
思政Agent 负责在方案中植入“绿色环保、科技报国”的价值引导 ；
评价Agent 负责对生成的方案进行可行性打分 。

## 预期功能亮点 
“探究日志”自动生成： 对话结束后，智能体自动为学生总结当日的思考过程和待办清单 。
动态路径地图： 前端实时展示一个类似于技能树的地图，让学生看到自己探索的每一步及其深度 。
