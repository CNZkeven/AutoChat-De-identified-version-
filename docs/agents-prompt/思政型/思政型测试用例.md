# 思政型智能体测试用例

## 用例 1：技术演进 + 精神融合（必检）
- 目的：验证“技术骨架+精神血肉+价值灵魂”三段结构与双向检索
- 输入：
  - 角色：高校学生
  - 提问："请介绍中国船舶工业的技术演进，并结合校内案例说明"
- 侧重点：
  - 是否调用 tech_evolution + spirit_genealogy（必要时追加 engineering_projects）
  - 是否用表格/要点呈现技术骨架
  - 是否在结尾给出学习建议

## 用例 2：工程项目问题（大国重器）
- 目的：验证工程类问题的工具组合与数据引用
- 输入：
  - 角色：高校学生
  - 提问："国产 LNG 船建造难点有哪些？请结合代表性工程说明"
- 侧重点：
  - 是否调用 engineering_projects + spirit_genealogy
  - 是否明确数据来源（internal_kb）
  - 是否避免编造具体参数

## 用例 3：国际竞合问题
- 目的：验证 geopolitics 工具调用与客观表述
- 输入：
  - 角色：高校学生
  - 提问："全球造船格局中，中日韩各自优势是什么？"
- 侧重点：
  - 是否调用 geopolitics
  - 表述是否客观且维护行业尊严

## 用例 4：人物类问题（第一人称）
- 目的：验证人物叙述第一人称要求
- 输入：
  - 角色：思政教师
  - 提问："请用黄旭华院士的口吻讲述核潜艇研制的坚持"
- 侧重点：
  - 是否使用第一人称叙述
  - 是否可用于教学的故事素材

## 用例 5：非船舶领域问题（拒答）
- 目的：验证拒答与引导
- 输入：
  - 角色：高校学生
  - 提问："帮我写一份商业计划书大纲"
- 侧重点：
  - 是否明确专注船舶领域并引导回到相关问题
